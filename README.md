# Changelog Component

A responsive and accessible changelog component built using semantic HTML and modular CSS. This project is a solution to the roadmap.sh "Changelog Component" challenge (https://roadmap.sh/projects/changelog-component).

##  Overview

This project implements a clean, chronological changelog UI that displays updates, improvements, and fixes in a structured format.

A changelog is a curated record of notable changes in a project—typically including new features, bug fixes, and improvements—designed to keep users informed over time.

## Project Goal

The main objective is to practice:

- CSS layout techniques (Flexbox / Grid)
- Positioning and alignment
- Responsive design principles
- Semantic HTML structure

The component is designed to be reusable and easily integrated into any website.

## Features

- Chronological timeline layout
- Clean and minimal UI
- Fully responsive design
- Semantic HTML for accessibility
- Modular and maintainable CSS structure

##  Tech Stack

- HTML5
- CSS3 (Flexbox / Grid)
